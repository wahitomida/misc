# ポートフォリオ要約（副業・案件受注向け）

## プロフェッショナルサマリー

AI と検索技術を組み合わせた実用的なプロダクト開発を得意とするエンジニアです。FastAPI を用いた Web API、LLM を活用した生成・要約パイプライン、ベクトル検索／RAG（Retrieval-Augmented Generation）や GraphRAG を含むナレッジグラフ連携の実装まで、要件定義からプロトタイピング、デプロイ、運用まで一貫して対応します。副業では短納期で価値を出すプロトタイプ提供、既存システムへの LLM 統合、検索基盤の設計・評価などを柔軟に受託します。

## 強み・こだわりポイント

- **実用重視**: PoC ではなく、運用を見据えた設計（API 仕様、認証、エラーハンドリング、監視）を優先します。
- **LLM と検索のハイブリッド設計**: プロンプト設計・コンテキスト管理・検索グラウンディングを組み合わせて誤生成を抑える実装を行います。
- **データ設計の徹底**: スキーマ化、メタデータ付与、データ品質チェックを導入し、検索性能や説明可能性を高めます。
- **自動化とテスト**: 単体テスト、統合テスト、CI によるデプロイ自動化、Infrastructure as Code（コンテナ化・Terraform 等）で再現性を担保します。
- **セキュリティ配慮**: 秘密情報の管理、アクセス制御、最小権限原則を重視します。

## 技術スタック（抜粋）

- 言語: Python (FastAPI, Pydantic), JavaScript(必要時)
- LLM / Embeddings: OpenAI 系 / Gemini 等のモデル、埋め込み（SentenceTransformers など）
- 検索・DB: Pinecone / FAISS / Milvus / Neo4j（グラフ）
- インフラ: Docker, GitHub Actions / CI, Terraform（提案ベース）
- その他: BeautifulSoup/Playwright（スクレイピング）、Elasticsearch（場合に応じて）

## 代表プロジェクト（抜粋）

1. Exhibit
    - 役割: 要件設計、FastAPI 実装、LLM と検索の統合、レポート生成パイプライン設計
    - 技術: FastAPI, Jinja2, LLM（生成モデル）, 検索グラウンディング

2. RAG 実験基盤
    - 役割: Neo4j を活用した知識グラフ設計、ベクトル検索の比較評価、GraphRAG の評価
    - 技術: Neo4j, 実験スクリプト、評価指標の自動集計

3. Web Search ツール
    - 役割: 検索結果の正規化・要約パイプライン実装、クロール・スクレイピングパイプライン設計
    - 技術: 検索 API 統合、データクレンジング、LLM による要約

## 副業で提供できる主な成果物

- 要件定義からの 1〜2 週間のプロトタイプ（API + 簡易 UI + テスト）
- 既存サービスへの LLM 統合（検索グラウンディング、応答品質改善）
- RAG/GraphRAG の評価レポートと改善提案
- データ収集・構造化パイプライン、埋め込み作成と検索インデックス構築
- コードレビュー、技術相談、ドキュメント整備・引き渡し

## 開発フロー・コミュニケーション

- スプリント単位（週次 or 2 週）での短いデモとフィードバックを重視します。
- GitHub を中心とした PR ベースの開発、単体テストと CI で品質を担保します。
- ドキュメント（README、設計メモ）を必ず用意し、運用時の引き継ぎを行います。

## 品質・セキュリティ対策

- 機密情報は環境変数・シークレットマネージャで管理し、公開リポジトリに残さない運用を徹底します。
- ロギングとメトリクスで挙動を観測し、誤生成や性能劣化を早期に検出します。

## 料金感・稼働目安

プロジェクトの範囲により変動しますが、短期プロトタイプはスポットで受託可能です。稼働は週 5–15 時間程度から対応できます。詳細は要件を伺ったうえで見積もりします。
